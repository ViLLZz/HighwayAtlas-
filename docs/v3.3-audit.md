# v3.3 Audit

This document records the 3.3 audit pass for reasoning, data presentation, grammar, and public readability.

## Goal

The goal of 3.3 is simple:

- make the atlas easier to trust
- make the numbers easier to understand
- make the writing sound like it was written for people, not for a model or a debug console

## Audit areas

### 1. Reasoning audit

The atlas should explain what a number means, not just show the number.

Examples of the direction in 3.3:

- prefer `Open now` over `Operational length`
- prefer `What to show` over `Status and visibility`
- prefer `How solid the data is` over `Data accuracy`

### 2. Data presentation audit

Several labels were technically correct but too internal for a public-facing atlas.

The 3.3 pass moves the wording toward:

- `Sections with official source`
- `Partly verified sections`
- `Estimated sections`
- `Share with official source`

That keeps the same data model while making it easier to understand what the atlas is claiming.

### 3. Grammar audit

The 3.3 pass cleaned up wording that sounded awkward, overly compressed, or too tool-like.

Examples:

- replaced phrases like `Awaiting verification` with `Needs checking`
- removed wording like `styled info panel`
- simplified long explanatory notes into shorter direct sentences

### 4. Statistics audit

The atlas still presents the same core figures, but the labels are now easier to scan:

- `Routes in atlas`
- `Sections in atlas`
- `Open now`
- `Under construction`
- `Still planned`
- `Network complete`

The intention is to make the first reading intuitive even if the viewer has never seen the project before.

## What changed in practice

1. atlas BG/EN interface strings were revised for clarity
2. source-quality labels were rewritten so they match what the filters actually do
3. selection and helper copy was simplified
4. playback and map notes were made more readable
5. console export summary labels were rewritten in plain English
6. README wording now reflects the clarity audit direction

## What stays the same

3.3 does not change the underlying route data model or provenance model.

It changes how the existing information is explained.

That distinction matters because a readability audit should not pretend to be a data update.

## Next audit candidates

- replace generic provenance wording with route-specific plain-language notes where possible
- review Bulgarian phrasing with a final native-language polish pass
- add short user-facing tooltips for source confidence and route status terms
