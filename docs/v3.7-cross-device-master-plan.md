# Motorway Atlas v3.7 Cross-Device Master Plan

This release line upgrades the atlas from a responsive interface into an explicit multi-profile product for Mac, iPad, and iPhone Safari.

## Goals

- Make desktop, tablet, and phone feel intentionally designed rather than width-shrunk variants.
- Stabilize iOS Safari viewport behavior during scroll, toolbar collapse, and orientation changes.
- Improve thumb reach, first-screen clarity, and tap confidence on iPhone.
- Keep the map visually dominant while preserving selection context and storytelling.
- Maintain one exporter code path and one canonical HTML artifact.

## 30-Step Program

1. Audit all current device profiles and identify accidental breakpoint overlap.
2. Promote device profiles into first-class runtime states.
3. Harden iOS visual viewport handling.
4. Replace static viewport assumptions with dynamic app-height variables.
5. Preserve Bulgaria-first framing through viewport transitions.
6. Reduce duplicated interface density on phone.
7. Create a dedicated phone-first map flow.
8. Add thumb-reach action zones for primary mobile actions.
9. Make reset and focus actions always easy to reach on mobile.
10. Improve quick-filter ergonomics for coarse pointers.
11. Tune safe-area usage for iPhone notch and home-indicator devices.
12. Refine iPad cockpit symmetry and first-screen hierarchy.
13. Keep desktop in a stronger editorial command-center layout.
14. Add profile-aware visual cues so users understand the current mode.
15. Strengthen type hierarchy per device class.
16. Normalize box sizing and internal spacing across cards.
17. Improve horizontal scroller affordances for route and filter rails.
18. Reduce visual noise in passive states.
19. Improve focus and keyboard visibility states.
20. Add reduced-motion safeguards for accessibility and calmer playback.
21. Improve animation timing so motion feels deliberate, not generic.
22. Keep side panels from fighting the map on smaller screens.
23. Improve map-stage composition in portrait vs landscape.
24. Add stronger information scent for notes, routes, and lot tabs on mobile.
25. Improve generated-at and data-confidence readability.
26. Document the release as v3.7 in project docs.
27. Expand smoke tests for new device-profile markers.
28. Run unit tests and alpha verification.
29. Regenerate atlas outputs for Pages.
30. Push the release to GitHub for live device testing.

## Planned v3.7 Deliverables

- Dedicated phone action dock.

- Visual-viewport-backed app-height runtime updates.
- Reduced-motion mode support.
- Refined desktop, tablet, and phone profile tuning.
