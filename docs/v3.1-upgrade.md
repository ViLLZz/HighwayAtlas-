# v3.1 Upgrade

This document captures the current 3.1 upgrade line for the Bulgarian Motorway Atlas.

## Delivered in 3.1

- explicit split between official records and secondary narrative references in the domain model
- provenance diagnostics exported beside geometry and continuity diagnostics
- atlas UI evidence grading for section-level data trust
- visible ownership and usage notice in the hosted/static atlas
- GitHub-backed Pages deployment already updating automatically on every push to `main`

## What changed technically

- `RouteSegment` now carries:
  - `OfficialSourceName`
  - `OfficialSourceUrl`
  - `OfficialSourceKind`
  - `OfficialSourceVerifiedOn`
  - `SecondarySourceName`
  - `SecondarySourceUrl`
- `RouteEngine` now emits provenance diagnostics in addition to geometry and continuity diagnostics
- `Program.cs` exports provenance into `exports/atlas/network-diagnostics.json`
- `HtmlAtlasExporter.cs` shows:
  - evidence grade
  - official record
  - supporting reference
  - verification date
  - ownership notice

## Official-source standard

The target standard for 3.1 is:

1. each section has at least one official source
2. generic official sources are acceptable only as a temporary floor
3. route-specific official pages, bulletins, or notices are preferred over generic portal links
4. secondary references remain supporting context only

## Remaining backlog

- replace generic API Bulgaria bulletin entrypoints with route-specific official URLs wherever available
- continue per-corridor verification for opening years, forecast years, and lot boundaries
- introduce structured official source catalogs per corridor instead of relying on seed-level defaults
- add GIS-grade route geometry where official or authoritative public geodata can be validated

## Definition of better data in 3.1

The goal is not to hide uncertainty. The goal is to make uncertainty explicit and measurable:

- official evidence is shown separately from narrative context
- modeled spans are still visible, but clearly marked
- diagnostics tell you where the official-source coverage is still weak

That is a stronger product foundation than presenting mixed-confidence data as if all sections were equally verified.
