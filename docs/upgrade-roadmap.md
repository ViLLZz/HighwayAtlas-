# Motorway Atlas Upgrade Roadmap

## Vision

Transform the current local preview into a bilingual motorway intelligence platform for Bulgaria with premium interaction design, stronger geographic fidelity, and source-backed infrastructure storytelling.

## Phase 1 — Delivered in this iteration

- bilingual BG/EN atlas shell
- refined corridor geometry with intermediate route points
- Tesla-inspired dark infographic layout
- multiple basemap styles
- animated corridor focus and section inspector
- richer route metadata model
- Hemus delay story block
- one-click local launcher

## Phase 2 — High-priority next work

### 1. Authoritative geometry
- import exact alignments from OSM / Overpass and official project sources
- split corridors into real lots and subsections
- validate kilometer lengths against official publications
- store source confidence per section

### 2. Verified project metadata
- budget history per lot
- forecast completion dates
- contractor and supervision entities
- source links and last verification timestamps
- delay reason taxonomy

### 3. Stronger infographic layer
- national completion dashboard
- regional impact panels
- corridor comparison by year and status
- travel time saved scenarios
- investment and delivery gap charts

### 4. Frontend platform step-up
- migrate from single exported HTML to a dedicated React / Next.js frontend
- move data into API-backed JSON endpoints
- support deep linking, bookmarks, and shareable route views
- add responsive mobile layout and presentation mode

## Phase 3 — Data credibility workflow

- introduce source registry model
- add manual review checklist for every edited section
- mark unverified fields clearly
- create update cadence for major corridors

## Phase 4 — Premium product finish

- cinematic transitions and polished motion system
- 3D terrain experiments for mountain corridors
- storytelling mode for Hemus, Struma, and Vidin–Botevgrad
- accessibility and typography review in both languages
- public-release grade QA pass

## Delivery principle

No speculative budgets or deadlines should be shown as factual. Fields may exist in the UI before population, but values should only be published after source verification.
