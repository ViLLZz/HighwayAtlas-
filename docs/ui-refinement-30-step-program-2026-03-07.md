# Motorway Atlas UI Refinement — Custom 30-Step Program

This is the focused 30-step program for taking the atlas to a sleeker, denser, more Apple-like dashboard quality while keeping motorway status clarity high.

## Layout architecture

1. Put the map in the primary top-right dashboard position.
2. Eliminate large empty bands between major modules.
3. Convert the main page into a tighter two-column desktop system.
4. Keep the map and inspector visually dominant above secondary content.
5. Make the left rail feel like a compact mission-control sidebar.
6. Move secondary summary cards below the map instead of above it.

## Navigation and tab system

7. Turn route pills into a single organized glass control shelf.
8. Group language and basemap actions into one clean control cluster.
9. Reduce visual noise in filter controls.
10. Make active tabs brighter and more tactile.
11. Make inactive tabs quieter but still legible.
12. Keep route, status, and language controls visually related.

## Liquid-glass visual language

13. Increase blur, edge highlights, and translucency carefully.
14. Add subtle inset highlights to primary controls.
15. Use layered glass backgrounds instead of flat dark panels.
16. Improve shadows so cards float without looking heavy.
17. Keep borders thin and luminous rather than harsh.
18. Preserve readability despite the glass treatment.

## Map and information density

19. Increase map prominence without breaking dashboard balance.
20. Keep lot interactions fast and visually precise.
21. Replace oversized lot numbers with cleaner premium markers.
22. Show non-open sections with clickable on-map status chips.
23. Ensure unfinished sections always expose their lot breakdowns.
24. Keep built, planned, and under-construction states visually distinct.

## Sidebar and supporting panels

25. Stack left-side modules in a more intentional reading order.
26. Reduce whitespace inside side panels.
27. Keep route and lot lists aligned with the selected map context.
28. Make headlines and support cards feel connected to the map, not detached from it.
29. Revalidate responsive behavior after each layout change.
30. Rebuild and export after every major refinement pass.
