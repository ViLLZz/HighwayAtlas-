# v3.2.3 Upgrade

This document records the current 3.2.3 release line for the Bulgarian Motorway Atlas.

## Release theme

3.2.3 is the atlas-usability release:

- the map had already become responsive in 3.2
- 3.2.3 makes the interaction model cleaner, more symmetrical, and easier to operate on iPad, iPhone, Android, and desktop
- the release keeps the 3.1 provenance model intact while improving the actual product feel of the generated atlas

## Delivered in 3.2.3

- premium display/body typography pairing using Sora and Manrope
- safe-area-aware shell spacing for phones and tablets
- more balanced topbar, headline strip, floating card, and panel proportions
- higher-fidelity map presentation with promoted satellite preset and stronger tile buffering
- frame-batched rendering with `requestAnimationFrame` for smoother state updates
- responsive map-fit padding for route, segment, and lot focus states
- top-of-map stage quick filters on compact layouts so essential status controls are available immediately without sidebar scrolling
- stronger visual hierarchy across cards, facts, notes, and timeline elements

## 20-step revitalization session

1. audited the atlas exporter layout to identify the mobile bottlenecks instead of treating them as isolated CSS bugs
2. upgraded the font system from a single utilitarian font to a display/body pairing better suited for an atlas product
3. introduced shared shell spacing variables to normalize rhythm across viewport sizes
4. made the shell safe-area aware so iPad and iPhone edges do not crowd the UI
5. normalized map height and minimum height through variables rather than scattered fixed values
6. widened route pills and improved their scroll behavior so they remain usable as actual controls on touch devices
7. tightened the desktop sidebar-to-map proportion to reduce visual imbalance in wide layouts
8. reworked headline-strip spacing and card framing to improve symmetry and reading order
9. strengthened floating-card styling so the map overlays feel intentional rather than merely translucent
10. promoted the satellite basemap into the featured set for richer atlas presentation
11. tuned canvas renderer padding and tile buffering for cleaner large-map interaction
12. replaced fixed map-fit paddings with viewport-aware paddings for default, route, segment, and lot focus states
13. batched redraw work behind `requestAnimationFrame` to avoid repeated immediate rerenders during interaction bursts
14. added resize and orientation rerender handling so compact devices recover layout cleanly after rotation
15. refined selection-panel hierarchy with better title balance, fact-card rhythm, and note treatment
16. improved timeline aesthetics with a clearer vertical guide and spacing discipline
17. tightened Leaflet chrome styling so controls and popups match the atlas visual language
18. surfaced the essential stage filters directly above the map on compact layouts to remove the need to scroll into the sidebar first
19. extended exporter smoke coverage so the new responsive and design markers are guarded by tests
20. re-ran the full verification flow so the GitHub Pages build can publish a technically clean release

## Why this matters

Before 3.2.3, the atlas had strong data and a capable interface, but smaller devices still inherited too much desktop behavior.
The practical problem was not only how the atlas looked. It was how quickly a user could operate it when opening the hosted page on a phone or tablet.

The most visible fix is the stage quick-filter dock at the top of the map on compact layouts. That change removes an immediate usability penalty on iPad and other mobile devices, because the user no longer has to scroll into the sidebar just to switch between open, under-construction, and planned views.
